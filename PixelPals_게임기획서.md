# PixelPals — 게임 기획서 v2.0

> **나만의 픽셀 캐릭터를 그리고, 가챠로 세상에 풀고, 모험으로 모아라**

---

## 1. 게임 개요

**장르:** 소셜 수집형 / 크리에이터 가챠 / 샌드박스
**플랫폼:** 웹 브라우저 (1차) → 모바일 PWA (2차) → 네이티브 앱 (3차)
**아트 스타일:** 픽셀아트 (16x16 ~ 32x32 캐릭터, 타일맵 월드)
**타겟 유저:** 캐주얼 게이머, 일러스트/픽셀아트 크리에이터, 팬덤 커뮤니티 (10대~30대)
**핵심 키워드:** UGC 가챠 + 크리에이터 이코노미 + 수집 & 모험

### 한 줄 요약

"누구나 자신만의 픽셀 캐릭터를 그려 가챠 머신에 넣고, 사람들은 그 캐릭터를 뽑고 키우며, 던전과 채집으로도 모을 수 있는 크리에이터 중심 수집형 게임"

### 핵심 경험

크리에이터는 픽셀 에디터로 캐릭터를 그리고, 자기 공간에 가챠 머신을 세팅한다. 희귀도, 확률, 연출까지 모두 크리에이터가 설정한다. 방문자는 인게임 화폐로 캐릭터를 뽑는데, 같은 캐릭터라도 매번 성격과 스탯이 랜덤으로 결정된다. 뽑기만이 유일한 경로는 아니다. 던전 탐험, 채집, 낚시 등의 모험 콘텐츠에서도 다른 크리에이터의 캐릭터를 낮은 확률로 획득할 수 있다. 좋아하는 유튜버나 작가가 그린 캐릭터를 뽑고 수집하는 팬 경험이 핵심 재미 요소다.

---

## 2. 핵심 게임 루프

```
크리에이터 루프:                    플레이어 루프:
                                   
  그린다 → 가챠 세팅 →              방문 → 뽑기 → 수집
    ↑        공간 꾸미기               ↑        ↓
    │            ↓                    │     모험/던전
    └──── 수익/명성 ←── 방문자 유입 ←─┘     으로도 획득
```

### 크리에이터 (공급자)

| 단계 | 설명 |
|------|------|
| **그리기 (Draw)** | 픽셀 에디터로 캐릭터, 가구, 아이템 제작 |
| **세팅 (Setup)** | 가챠 테이블 구성 — 희귀도, 확률, 연출 설정 |
| **꾸미기 (Decorate)** | 자기 공간을 꾸며서 방문자를 유치 |
| **수익 (Earn)** | 방문자가 뽑기할 때마다 인게임 화폐 수익 |

### 플레이어 (수요자)

| 단계 | 설명 |
|------|------|
| **탐방 (Visit)** | 크리에이터 공간을 방문하고 둘러보기 |
| **뽑기 (Gacha)** | 마음에 드는 가챠 머신에서 캐릭터 뽑기 |
| **모험 (Adventure)** | 던전/채집/낚시로 캐릭터를 획득하는 대안 루트 |
| **수집 (Collect)** | 뽑은 캐릭터를 내 공간에 배치하고 도감 채우기 |

## 3. 캐릭터 시스템

### 3.1 캐릭터 생성 — 픽셀 에디터

이 게임에서 가장 중요한 기능. 그리기 도구의 품질이 곧 게임의 품질이다.

**에디터 핵심 기능:**

- **캔버스 크기:** 16x16 (기본), 24x24, 32x32 선택 가능
- **팔레트:** 기본 16색 팔레트 제공 + 커스텀 팔레트 생성/저장/공유
- **레이어:** 몸체 / 얼굴 / 악세서리 / 이펙트 (4레이어, 레이어별 표시/숨김)
- **도구:** 연필, 지우개, 채우기, 선, 사각형, 원, 스포이드, 선택/이동
- **편의 기능:** 실행취소/다시실행 (무제한), 좌우 대칭 모드, 복사/붙여넣기, 그리드 토글
- **애니메이션:** 프레임 단위 편집, 어니언 스킨(이전 프레임 반투명 표시), 실시간 미리보기
- **템플릿:** 초보자를 위한 기본 템플릿 (고양이, 강아지, 로봇 등)
- **에셋 종류:** 캐릭터 / 가구 / 음식 / 악세서리 / 타일 / 배경 오브젝트

**기술 구현 — 드로잉 엔진:**

| 후보 | 장점 | 단점 | 판정 |
|------|------|------|------|
| **Konva.js** | 레이어 내장, 터치 지원, 이벤트 처리 우수, 8000객체@23fps | 범용 라이브러리라 픽셀 특화 아님 | 권장 (메인) |
| **Raw Canvas 2D** | 최고 성능, 오버헤드 제로, 픽셀 그리드에 최적 | 터치/레이어 직접 구현 필요 | 대안 (고성능) |
| Piskel.js | 검증된 픽셀 에디터 | 확장성 제한, 커스터마이징 어려움 | 참고용 |
| Fabric.js | 풍부한 API | 무겁고 느림 (8000객체@9fps) | 부적합 |

**권장 구성:** Konva.js를 베이스로 레이어/이벤트 처리를 맡기고, 실제 픽셀 렌더링은 OffscreenCanvas로 최적화. 모바일은 Pointer Events API로 터치/마우스 통합 처리.

### 3.2 캐릭터 속성 — 크리에이터가 정하는 것 vs 랜덤

```
캐릭터 설계서 (크리에이터가 정함)
├── 비주얼: 스프라이트 시트 (픽셀 에디터로 제작)
├── 이름: 캐릭터 이름
├── 희귀도: 크리에이터가 직접 지정 (커스텀 등급 가능)
└── 설명: 캐릭터 배경 스토리, 한마디 등

뽑기 결과 (매번 랜덤으로 결정)
├── 성격: 활발한 / 수줍은 / 지적인 / 장난꾸러기 / 다정한 (랜덤)
├── 스탯 (각 1~100 랜덤)
│   ├── 힘 (STR): 던전 전투력에 영향
│   ├── 민첩 (DEX): 채집/낚시 성공률에 영향
│   ├── 지능 (INT): 퀘스트 보상에 영향
│   ├── 매력 (CHA): 다른 캐릭터와의 교류 효과에 영향
│   └── 행운 (LUK): 드롭률, 크리티컬 등에 영향
├── 특성 (1~2개 랜덤 부여)
│   예: "아침형 인간", "미식가", "겁쟁이", "수집광", "낚시 천재" 등
└── 외형 변이 (선택적)
    크리에이터가 설정한 경우, 색상 변이(컬러 팔레트 랜덤 교체) 적용
```

같은 "레전드 고양이" 캐릭터를 뽑아도 STR 95의 전투형이 나올 수도 있고 LUK 99의 행운형이 나올 수도 있다. 이 때문에 "이 조합이 나올 때까지" 다시 뽑을 동기가 생긴다.

### 3.3 캐릭터 움직임 — 코드 기반 기본 애니메이션

크리에이터가 프레임 애니메이션을 만들 수도 있지만, 기본적으로 다음 움직임은 코드로 자동 처리된다.

**기본 제공 움직임:**

- **좌우 플립:** 이동 방향에 따라 스프라이트 자동 좌우 반전
- **통통 바운스:** 이동 시 Y축 사인파 적용으로 통통 튀는 느낌
- **아이들 흔들림:** 가만히 서 있을 때 미세한 좌우/상하 흔들림
- **등장 팝:** 새로 나타날 때 스케일 0→1.2→1.0 바운스 애니메이션
- **감정 이펙트:** 기쁨(하트 파티클), 슬픔(눈물 방울), 놀람(! 말풍선), 분노(연기)
- **상호작용:** 다른 캐릭터/오브젝트 근처에서 말풍선 표시
- **수면:** 눈 감기(프레임 교체 없이 어두운 오버레이) + zzZ 파티클

이 기본 애니메이션 세트만으로도 캐릭터가 살아 있는 느낌을 준다. 크리에이터가 프레임을 추가로 그리면 더 풍부한 움직임이 된다.

## 4. 가챠 시스템 — 핵심 시스템

이 게임의 가장 독특한 점: 가챠를 운영하는 주체가 게임사가 아니라 크리에이터다.

### 4.1 크리에이터의 가챠 설정

**가챠 머신 구성:**

크리에이터는 자기 공간에 가챠 머신을 설치하고 다음을 설정한다.

- **등록 캐릭터:** 자신이 그린 캐릭터 중 가챠에 넣을 캐릭터 선택
- **희귀도 등급:** 완전 커스텀 가능

```
예시 A (클래식 스타일):
  ★ 일반 (Common) — 60%
  ★★ 희귀 (Rare) — 25%
  ★★★ 영웅 (Epic) — 12%
  ★★★★ 전설 (Legendary) — 2.5%
  ★★★★★ 신화 (Mythic) — 0.5%

예시 B (커스텀 스타일 — "레바의 고양이 뽑기"):
  🐱 노란 고양이 — 40%
  🐱 검은 고양이 — 30%
  🐱 삼색 고양이 — 20%
  🐱 무지개 고양이 — 8%
  🐱 우주 고양이 — 2%
```

- **뽑기 비용:** 1회당 인게임 화폐 가격 설정 (최소/최대 범위 내)
- **천장 시스템:** 선택적 — N회 이내 최고 등급 보장 (크리에이터가 설정)
- **가챠 연출:** 뽑기 애니메이션 커스터마이징 (배경색, 이펙트, 사운드)
- **확률 공시:** 모든 확률은 투명하게 공개 (필수 규칙)

### 4.2 뽑기 경험

1. 방문자가 크리에이터의 공간에 있는 가챠 머신에 접근
2. 확률표를 확인하고, 인게임 화폐를 투입
3. 뽑기 애니메이션 재생 (크리에이터가 설정한 연출)
4. 캐릭터 등장 + 랜덤 스탯/성격 공개
5. "오오! STR 97 전설 고양이다!" → 수집/공유

### 4.3 뽑기 외 획득 경로 — 모험 시스템

뽑기만으로 캐릭터를 얻는 것은 불공평하다. 시간과 노력으로도 캐릭터를 모을 수 있는 대안 경로를 제공한다.

**던전 탐험:**
- 자동/반자동 전투 시스템 (소유한 캐릭터 파티 구성)
- 던전 클리어 보상으로 랜덤 캐릭터 조각 획득
- 조각을 일정 수 모으면 캐릭터 완성 (예: 10조각 = 일반, 50조각 = 전설)
- 던전 난이도에 따라 획득 가능한 캐릭터 등급 상승

**채집 & 낚시:**
- 자기 공간 또는 공용 맵에서 채집/낚시 미니게임
- 확률적으로 캐릭터 알(Egg) 획득
- 알을 부화시키면 랜덤 크리에이터의 캐릭터 등장
- 채집 레벨이 높을수록 희귀 알 확률 상승

**퀘스트 보상:**
- 일일/주간 퀘스트 완료 시 뽑기 티켓 지급
- 특별 이벤트 퀘스트에서 한정 캐릭터 직접 획득

**교환 시스템:**
- 플레이어 간 캐릭터 1:1 교환
- 중복 캐릭터를 "조각"으로 분해 → 상점에서 원하는 캐릭터 조각 교환

### 4.4 가챠 경제 밸런스

| 항목 | 설정 |
|------|------|
| 일일 무료 화폐 지급 | 뽑기 2~3회 분량 |
| 던전/채집 일일 캐릭터 획득 | 일반 1~2체, 희귀 이상 0~1체 |
| 천장 시스템 | 크리에이터 설정 (권장 100회 이내) |
| 중복 캐릭터 활용 | 합성으로 스탯 보너스 또는 조각 분해 |

## 5. 공간 시스템

### 5.1 나만의 공간 — 크리에이터의 무대

모든 유저에게 기본 제공되는 개인 공간. 크리에이터에게는 자신의 가챠를 운영하는 "가게"이자, 자기 세계관을 보여주는 "갤러리"이기도 하다.

**구성:**

- **기본 크기:** 16x16 타일 (확장 가능, 최대 64x64)
- **구역:** 실내(전시/가챠존) + 실외(정원/모험존)
- **타일맵 에디터:** 바닥, 벽, 가구, 장식물을 자유롭게 배치
- **가챠 머신 배치:** 공간 내 원하는 위치에 가챠 머신 설치 (복수 가능)
- **테마:** 숲속 오두막 / 도시 아파트 / 해변 별장 / 우주 정거장 등

**소유 캐릭터 배치:**
수집한 캐릭터를 공간 곳곳에 배치하면, 캐릭터가 기본 애니메이션(바운스, 아이들 흔들림 등)으로 돌아다니며 공간을 살아 있게 만든다.

### 5.2 방문 시스템

**방문 방식:**

- 크리에이터 검색으로 직접 방문
- 인기/추천 공간 목록에서 방문
- 친구 목록에서 방문
- 초대 링크(URL) 공유로 방문

**방문 시 가능한 활동:**

- 가챠 머신에서 뽑기
- 공간 구경 & 방명록 남기기
- 크리에이터의 전시 캐릭터 감상
- "좋아요" 누르기 (크리에이터 랭킹에 반영)

### 5.3 공용 공간

- **광장:** 캐릭터들이 돌아다니며 다른 유저와 마주침
- **던전 입구:** 파티를 구성하고 던전에 입장
- **채집 필드:** 낚시터, 광산, 숲 등 채집 미니게임
- **거래소:** 캐릭터 교환, 아이템 거래
- **갤러리:** 주간 인기 크리에이터/캐릭터 전시

## 6. 크리에이터 이코노미 — 팬게임 생태계

### 6.1 타겟 크리에이터

이 게임의 성장은 크리에이터 유입에 달려 있다. 핵심 타겟:

- **그림 유튜버/스트리머:** 레바, 침착맨, 우왁굳 팬게임 등 — 자신의 캐릭터를 팬들이 뽑는 경험
- **픽셀아트 아티스트:** 자신의 작품을 게임 내 생태계로 배포
- **일반 유저:** 간단한 템플릿 기반으로 누구나 크리에이터 가능

### 6.2 크리에이터 보상 구조

| 수익원 | 설명 |
|--------|------|
| **뽑기 수수료** | 방문자가 가챠할 때마다 투입 화폐의 일정 비율 수령 |
| **방문자 보너스** | 일일 방문자 수에 따른 추가 보상 |
| **인기 랭킹 보상** | 주간/월간 인기 크리에이터 랭킹 상위 보상 |
| **크리에이터 등급** | 뉴비 → 인기 → 스타 → 레전드 (등급별 추가 기능 해금) |

### 6.3 크리에이터 보호

- 타 크리에이터의 캐릭터를 무단 복제/도용 시 신고 시스템
- 원본 크리에이터 표시 (모든 캐릭터에 제작자 워터마크)
- AI 기반 유사도 검사로 도용 자동 탐지

## 7. AI 시스템 — 백엔드 안전망

이 게임에서 AI는 전면이 아니라 뒷단에서 안전하고 건강한 생태계를 유지하는 역할을 한다.

### 7.1 콘텐츠 심사 AI

- **선정성 필터링:** 업로드되는 모든 스프라이트를 AI가 자동 검수
- **폭력/혐오 검출:** 부적절한 상징, 텍스트 포함 여부 검사
- **저작권 유사도 검사:** 기존 등록 캐릭터와의 유사도 분석, 도용 방지
- **등급 자동 분류:** 전체이용가 / 12세 이상 자동 태깅

### 7.2 스팸/어뷰징 탐지 AI

- **봇 계정 탐지:** 비정상적 행동 패턴 (반복 뽑기, 자동 클릭) 감지
- **경제 어뷰징 감시:** 화폐 복제, 비정상 거래 패턴 탐지
- **던전 매크로 탐지:** 행동 타이밍 분석으로 자동화 스크립트 차단
- **가챠 확률 검증:** 크리에이터가 설정한 확률이 실제로 적용되는지 통계 검증

### 7.3 AI 기술 스택

| 용도 | 기술 | 비용 |
|------|------|------|
| 이미지 검수 | 경량 CNN 분류 모델 (자체 호스팅) | 낮음 |
| 유사도 검사 | 이미지 임베딩 + 코사인 유사도 | 중간 |
| 행동 분석 | 룰 기반 + 이상 탐지 모델 | 낮음 |
| 텍스트 필터링 | 금칙어 사전 + 경량 NLP | 낮음 |

## 8. 보안 설계 — 웹 게임의 과제와 해법

웹 게임은 클라이언트가 100% 노출되므로, "클라이언트를 절대 신뢰하지 않는" 서버 권위 모델(Server-Authoritative)이 필수다.

### 8.1 핵심 원칙: 서버가 모든 것을 결정한다

```
클라이언트                          서버
   │                                │
   │──── "뽑기 요청" ──────────────→│ 화폐 잔액 확인
   │                                │ 서버에서 난수 생성
   │                                │ 결과 결정 + DB 기록
   │←── "결과: 전설 고양이(STR 92)" │
   │                                │
   ✕ 클라이언트에서 결과 조작 불가   │
```

### 8.2 주요 위협과 대응

| 위협 | 대응 |
|------|------|
| **가챠 결과 조작 (DevTools)** | 모든 뽑기 연산은 서버에서만 수행. 클라이언트는 결과를 표시만 함 |
| **화폐 복제 / 트랜잭션 리플레이** | Nonce 기반 거래 ID + 서버 측 잔액 관리. 모든 거래는 atomic 처리 |
| **스프라이트 도용 (이미지 추출)** | 완벽한 방지는 불가. 대응: 저해상도 프리뷰 + 워터마크 + 법적 보호 |
| **던전/채집 봇** | 서버 측 행동 분석 (타이밍 분산 체크) + 비정상 패턴 탐지 + 레이트 리밋 |
| **계정 도용** | OAuth 2.0 소셜 로그인 + 2FA 지원 |
| **API 남용** | Rate Limiting + JWT 토큰 + IP 기반 접근 제어 |

### 8.3 웹 vs 네이티브 보안 비교 (검토 결과)

**결론: 웹으로 충분하다. 단, 서버 중심 아키텍처가 전제 조건.**

| 항목 | 웹 | 네이티브 |
|------|-----|---------|
| 코드 보호 | 난독화 (우회 가능) | 바이너리 (역공학 어렵지만 가능) |
| 보안 패치 배포 | 즉시 (서버 배포) | 스토어 심사 1~3일 |
| 안티치트 | 서버 검증 + 행동 분석 | 커널 수준 안티치트 가능 |
| 접근성 | URL만으로 접속 | 설치 필요 |
| 결론 | 가챠 결과가 서버 결정이면 충분 | 실제 현금 대규모 거래 시 고려 |

이 게임은 인게임 화폐 기반이고, 모든 게임 로직이 서버에서 처리되므로 웹으로 운영 가능하다. 추후 규모 확대 시 네이티브 앱 추가는 보안이 아닌 UX/마케팅 이유로 진행하면 된다.

```
┌──────────────────────────────────────────────────┐

└───────────────────┬──────────────────────────────┘
                    │ WebSocket + REST API
┌───────────────────▼──────────────────────────────┐
│                    서버 (권위 모델)                  │
│  Node.js + NestJS (API 서버)                      │
│  가챠 엔진: 서버 전용 (crypto.randomBytes 기반)     │
│  Socket.io (실시간: 공간 동기화, 방문자 표시)        │
│  Redis (세션, Nonce 관리, 레이트 리밋)              │
└───────────────────┬──────────────────────────────┘
                    │
┌───────────────────▼──────────────────────────────┐
│                    AI 계층 (백엔드)                  │

                    │

│                   데이터베이스                       │

│  S3 + CDN (스프라이트 시트, 에셋 이미지)            │

└──────────────────────────────────────────────────┘
```

### 9.2 가챠 서버 설계

```
POST /api/gacha/pull
├── 요청: { machineId, userId, nonce }
├── 서버 처리:
│   ├── 1. Nonce 유효성 검증 (중복 방지)
│   ├── 2. 화폐 잔액 확인 & 차감 (atomic)
│   ├── 3. 크리에이터 설정 확률 테이블 로드
│   ├── 4. crypto.randomBytes로 난수 생성
│   ├── 5. 캐릭터 결정 + 스탯/성격 랜덤 생성
│   ├── 6. 결과를 DB에 기록 (캐릭터 소유권 이전)
│   └── 7. 크리에이터에게 수수료 지급
└── 응답: { character, stats, personality, rarity, animation }
```

## 10. 수익 모델

기본 플레이 무료(F2P). 인게임 화폐는 플레이로도 충분히 획득 가능.

| 항목 | 설명 | 가격대 (예시) |
|------|------|--------------|
| **인게임 화폐 패키지** | 가챠/상점용 화폐 구매 | ₩1,000~50,000 |
| **공간 확장** | 16x16 → 32x32, 64x64 | ₩3,000~5,000 |
| **캐릭터 보관함 확장** | 기본 50칸 → 추가 | ₩2,000/50칸 |
| **프리미엄 에디터 기능** | 추가 레이어, 팔레트, 이펙트 | ₩3,000~5,000 |
| **시즌 패스** | 시즌 전용 보상 트랙 + 한정 가챠 | ₩10,000/시즌 |
| **가챠 수수료** | 크리에이터 가챠 거래 시 10% 플랫폼 수수료 | - |
| **크리에이터 프리미엄** | 가챠 머신 추가 설치, 고급 연출 | ₩5,000/월 |

**원칙:** 돈을 안 써도 매일 플레이하면 뽑기를 할 수 있고, 던전/채집으로도 캐릭터를 모을 수 있다. 과금은 "더 빠르게, 더 편하게"일 뿐.

## 12. 차별화 포인트

| vs | PixelPals의 차별점 |
|----|-------------------|
| **vs 로블록스** | 코딩 불필요, 그리기만으로 콘텐츠 제작. 가챠 시스템으로 수집 재미 |
| **vs 일반 가챠 게임** | 가챠 콘텐츠를 유저가 직접 만듦. 크리에이터가 확률과 연출을 설정 |
| **vs 다마고치** | UGC 생태계, 소셜 방문, 모험 콘텐츠 |
| **vs Character.AI** | AI 대화가 아닌, 수집과 꾸미기 중심. 크리에이터 경제 |
